## 1. 개요

### 1.1 한 줄 정의

로그인한 누구나 곡을 올리고, 모든 청취자가 자유롭게 듣는 공동체 공유 음악방.

### 1.2 채널 포지셔닝

| 채널 | 소스 | 청취 방식 | 조작 권한 |
|---|---|---|---|
| KBS 클래식FM / 극동방송 | HLS 외부 스트림 | 실시간 | 없음 |
| 말씀의 전당 / 찬양의 숲 | R2 개인 업로드 | 개인 재생 | 본인 |
| 봄소리방송 (예봄소리) | 관리자 송출 | 동기 청취 | 관리자 |
| **열린 음악방 (신규)** | **누구나 업로드** | **개인 재생 (1단계)** | **업로더 + 관리자** |

### 1.3 개발 단계

- **1단계 (본 문서 범위)**: 폴더 기반 목록 재생 — 찬양의 숲과 동일한 개인 재생 방식
- **2단계 (향후 확장)**: 랜덤 라이브 모드 — 전체 곡 셔플 동기 재생 (큐 기반)

## 2. 기본 폴더 구조

### 2.1 기본 폴더 (5개, 삭제 불가)

| 순서 | 폴더명 | 아이콘 | 용도 |
|---|---|---|---|
| 1 | 졸음운전방지 | ☀️ | 운전 중 각성용 활기찬 음악 |
| 2 | 온화한BGM | 🌿 | 잔잔한 배경 음악 |
| 3 | 묵상기도 | 🕯️ | 기도·묵상 시간용 |
| 4 | 성령충만 | 🔥 | 강렬한 찬양·워십 |
| 5 | 자유곡 | 🎵 | 분류 불가 전부 |

- 감성 강도순 배치: 활기찬(졸음운전방지) → 무관(자유곡)
- "자유곡"은 안전망 역할 — 분류 고민 없이 올릴 수 있는 곳

### 2.2 사용자 생성 폴더

- 로그인 사용자 누구나 생성 가능
- 폴더명: 2~10자, 동일 이름 중복 불가
- 전체 폴더 수 상한: 15개 (기본 5개 + 사용자 생성 최대 10개)
- 자동 정리: 곡 0개 상태가 30일 지속 시 Cron Trigger로 자동 삭제
- 기본 5개 폴더는 자동 삭제 대상에서 제외
- 관리자는 언제든 폴더 삭제·이름 변경 가능

## 3. 핵심 아키텍처: 물리 파일 1개, 논리 참조 N개

### 3.1 원칙

R2에는 파일을 한 번만 저장한다. KV의 폴더 메타데이터에서 같은 `r2Key`를 참조하는 방식으로, 한 곡이 여러 폴더에 존재할 수 있다.

```
R2 (물리 저장소)
└── openroom/files/
    └── abc123.mp3              ← 실제 파일은 1개

KV (논리 구조)
├── folder:졸음운전방지 → [..., {id:"ref-01", r2Key:"abc123.mp3", name:"활기찬 아침"}]
├── folder:성령충만     → [..., {id:"ref-02", r2Key:"abc123.mp3", name:"활기찬 아침"}]
└── folder:자유곡       → []
```

### 3.2 파일 마스터 레코드

파일별 마스터 레코드에 참조 목록을 관리한다. 모든 참조가 제거되었을 때만 R2에서 물리 삭제한다.

```json
// KV key: openroom:file:{r2Key}
{
  "r2Key": "abc123.mp3",
  "originalName": "활기찬 아침",
  "uploader": "김은혜",
  "uploaderId": "user-uuid-001",
  "duration": 215,
  "size": 4200000,
  "mimeType": "audio/mpeg",
  "refs": ["졸음운전방지", "성령충만"],
  "createdAt": "2026-03-09T08:30:00Z"
}
```

### 3.3 동작별 처리

| 동작 | KV 변경 | R2 변경 |
|---|---|---|
| **업로드** | 대상 폴더에 참조 추가, 파일 마스터 레코드 생성, `refs`에 폴더명 추가 | R2에 파일 저장 |
| **복사** (관리자) | 대상 폴더에 참조 추가, `refs`에 폴더명 추가 | 없음 |
| **이동** (관리자) | 원본 폴더에서 참조 제거 + 대상 폴더에 참조 추가, `refs` 갱신 | 없음 |
| **이 폴더에서 제거** | 해당 폴더에서 참조 제거, `refs`에서 폴더명 제거 | `refs`가 빈 배열이면 R2 삭제 |
| **완전 삭제** (관리자) | 모든 폴더에서 참조 제거, 파일 마스터 레코드 삭제 | R2 삭제 |

## 4. KV 데이터 구조

### 4.1 전체 키 맵

```
openroom:folders                    → 폴더 목록 (순서 포함)
openroom:folder:{folderName}        → 해당 폴더의 곡 참조 배열
openroom:file:{r2Key}               → 파일 마스터 레코드
openroom:settings                   → 관리자 설정
```

### 4.2 각 키 상세

#### `openroom:folders`


### 4.3 R2 경로 규칙

```
openroom/files/{r2Key}
```

- `r2Key`는 업로드 시 UUID 기반으로 생성 (예: `a1b2c3d4-e5f6.mp3`)
- 폴더 구조는 KV에서만 관리하며, R2에는 flat 구조로 저장
- 폴더와 무관하게 파일은 모두 `openroom/files/` 하위에 저장

## 5. Worker API 엔드포인트

### 5.1 폴더 관련

| Method | Path | 권한 | 설명 |
|---|---|---|---|
| `GET` | `/api/openroom/folders` | 전체 | 폴더 목록 조회 (곡 수 포함) |
| `POST` | `/api/openroom/folders` | 로그인 | 폴더 생성 |
| `PUT` | `/api/openroom/folders/:name` | 관리자 | 폴더 이름·아이콘 변경 |
| `DELETE` | `/api/openroom/folders/:name` | 관리자 | 폴더 삭제 (기본 폴더 제외) |

### 5.2 곡 관련

| Method | Path | 권한 | 설명 |
|---|---|---|---|
| `GET` | `/api/openroom/folders/:name/tracks` | 전체 | 폴더 내 곡 목록 조회 |
| `POST` | `/api/openroom/folders/:name/tracks` | 로그인 | 곡 업로드 (multipart) |
| `PUT` | `/api/openroom/tracks/:refId` | 본인/관리자 | 곡 이름 수정 |
| `DELETE` | `/api/openroom/tracks/:refId` | 본인/관리자 | 해당 폴더에서 곡 제거 |

### 5.3 관리자 전용

| Method | Path | 권한 | 설명 |
|---|---|---|---|
| `POST` | `/api/openroom/admin/copy` | 관리자 | 곡을 다른 폴더로 복사 (복수 선택 가능) |
| `POST` | `/api/openroom/admin/move` | 관리자 | 곡을 다른 폴더로 이동 |
| `DELETE` | `/api/openroom/admin/files/:r2Key` | 관리자 | 완전 삭제 (모든 폴더 + R2) |
| `GET` | `/api/openroom/admin/storage` | 관리자 | 전체 저장 용량 현황 |
| `PUT` | `/api/openroom/admin/settings` | 관리자 | 설정 변경 |

### 5.4 요청/응답 예시

#### 곡 업로드 `POST /api/openroom/folders/찬양/tracks`
```
Content-Type: multipart/form-data

file: (binary)
displayName: "주의 사랑이 나를 감싸네"    // 선택 — 미입력 시 파일명에서 추출
```

응답:
```json
{
  "success": true,
  "track": {
    "refId": "ref-uuid-003",
    "r2Key": "x7y8z9.mp3",
    "displayName": "주의 사랑이 나를 감싸네",
    "folder": "찬양",
    "duration": 225
  }
}
```

#### 곡 복사 `POST /api/openroom/admin/copy`
```json
{
  "r2Key": "abc123.mp3",
  "sourceFolder": "졸음운전방지",
  "targetFolders": ["성령충만", "자유곡"]
}
```

응답:
```json
{
  "success": true,
  "copied": ["성령충만", "자유곡"],
  "skipped": [],
  "refs": ["졸음운전방지", "성령충만", "자유곡"]
}
```

#### 곡 이동 `POST /api/openroom/admin/move`
```json
{
  "refId": "ref-uuid-001",
  "sourceFolder": "자유곡",
  "targetFolder": "묵상기도"
}
```

## 6. UI 화면 구성

### 6.1 메인 화면 — 폴더 목록

```
┌──────────────────────────────┐
│  🎵 열린 음악방        [로그인]│
├──────────────────────────────┤
│                              │
│  ☀️ 졸음운전방지 (3곡)     ▶  │
│  🌿 온화한BGM (12곡)      ▶  │
│  🕯️ 묵상기도 (7곡)        ▶  │
│  🔥 성령충만 (5곡)        ▶  │
│  🎵 자유곡 (2곡)          ▶  │
│  ─────────────────────────── │
│  ✨ 새벽기도 (4곡)         ▶  │  ← 사용자 생성 폴더 (구분선 아래)
│  ✨ 성가대 연습곡 (8곡)    ▶  │
│                              │
│  [ + 폴더 만들기 ]           │  ← 로그인 사용자에게만 표시
│                              │
├──────────────────────────────┤
│  ▶ advancement 지금 재생 중: 은혜 아니면  │  ← 미니 플레이어 (하단 고정)
└──────────────────────────────┘
```

**구현 노트:**
- 기본 폴더와 사용자 생성 폴더 사이에 시각적 구분선 삽입
- 각 폴더 옆에 곡 수 표시
- 비로그인 사용자: 폴더 탐색·재생은 가능, 업로드·폴더 생성 버튼 숨김
- 미니 플레이어: 현재 재생 중인 곡이 있을 때만 하단에 표시

### 6.2 폴더 내부 — 곡 목록

```

│  ← 🙏 찬양                   │
├──────────────────────────────┤
```

#### [⋮] 곡 옵션 메뉴 — 권한별 분기

**일반 사용자 (본인 곡):**
```

│  🗑️ 이 폴더에서 삭제      │
└─────────────────────────┘
```

**일반 사용자 (타인 곡):**
- 옵션 메뉴 없음 — [▶] 재생만 가능

**관리자:**
```
┌─────────────────────────┐
│  📋 다른 폴더에 복사      │
│  📦 다른 폴더로 이동      │
│  ✏️ 곡 이름 수정          │
│  🗑️ 이 폴더에서 제거      │
│  ⚠️ 완전 삭제 (모든 폴더)  │
└─────────────────────────┘
```

### 6.3 관리자 — 복사/이동 폴더 선택

"복사" 또는 "이동" 탭 시 표시되는 바텀시트:

```
┌─────────────────────────┐
│  복사할 폴더 선택    ✕   │
├─────────────────────────┤
│  ☀️ 졸음운전방지   ✓     │  ← 이미 존재 (비활성, 체크 표시)
│  🌿 온화한BGM     ☐     │  ← 선택 가능
│  🕯️ 묵상기도      ☐     │
│  🔥 성령충만      ☐     │
│  🎵 자유곡        ☐     │
│  ───────────────────── │
│  ✨ 새벽기도       ☐     │
├─────────────────────────┤
│  [ 복사 실행 ]           │  ← 1개 이상 선택 시 활성화
└─────────────────────────┘
```

- 체크박스 복수 선택 가능 (한 번에 여러 폴더로 복사)
- 이미 해당 곡이 존재하는 폴더는 ✓ 표시 + 비활성
- "이동"의 경우 단일 선택 (라디오 버튼)

### 6.4 곡 올리기 — 업로드 플로우

폴더 안에서 "곡 올리기"를 누르면 바텀시트 표시:

```
┌─────────────────────────────┐
│  곡 올리기                ✕  │
├─────────────────────────────┤
│                             │
│  ┌───────────────────────┐  │
│  │  📁 파일 선택하기      │  │  ← 탭하면 파일 피커
│  │  (MP3, M4A, 50MB 이하) │  │
│  └───────────────────────┘  │
│                             │
│  곡 이름                    │
│  ┌───────────────────────┐  │
│  │ 주의 사랑이 나를 감싸네  │  │  ← 파일명에서 자동 추출, 수정 가능
│  └───────────────────────┘  │
│                             │
│  ┌───────────────────────┐  │
│  │      ✅ 올리기         │  │
│  └───────────────────────┘  │
│                             │
│  업로드 중... ▰▰▰▰▰▱▱ 72%  │  ← 진행 표시
│                             │
└─────────────────────────────┘
```

**핵심 설계 원칙:**
- 이미 폴더 안에서 "올리기"를 눌렀으므로 **폴더 선택 단계 불필요**
- 곡 이름은 파일명에서 확장자 제거 후 자동 채움 → 수정 가능
- 필수 입력: 파일만. 곡 이름은 자동 채움이므로 수정 안 해도 됨
- 파일 선택 → 올리기 = **2탭 완료**

### 6.6 빈 폴더 상태

곡이 하나도 없는 폴더에 진입했을 때:

```
┌──────────────────────────────┐
│  ← 🕯️ 묵상기도               │
├──────────────────────────────┤
│                              │
│           🎵                 │
│   아직 곡이 없습니다          │
│   첫 번째 곡을 올려주세요!    │
│                              │
│      [ 🎵 곡 올리기 ]        │
│                              │
└──────────────────────────────┘
```

- "오류"가 아닌 "초대"의 톤
- 비로그인 시: "로그인하면 곡을 올릴 수 있습니다" + 올리기 버튼 숨김

## 8. 남용 방지 장치

사용자에게는 친절한 안내 문구만 노출하고, 수치는 관리자가 KV `openroom:settings`에서 조정한다.

| 장치 | 기본값 | 안내 문구 |
|---|---|---|
| 파일 크기 제한 | 50MB | "파일 크기가 50MB를 초과합니다" |
| 1인당 업로드 상한 | 20곡 | "곡은 최대 20개까지 올릴 수 있습니다" |
| 전체 폴더 수 상한 | 15개 | "폴더는 최대 15개까지 만들 수 있습니다" |
| 허용 파일 형식 | MP3, M4A, WAV | "MP3, M4A, WAV 파일만 올릴 수 있습니다" |
| 총 저장 용량 | 2GB | (관리자에게만 표시) |
| 빈 폴더 자동 삭제 | 30일 | (백그라운드 처리, 사용자 미고지) |

| 작업 | 주기 | 동작 |
|---|---|---|
| 빈 폴더 정리 | 1일 1회 | `isDefault: false` 이면서 곡 0개, 생성일로부터 30일 경과 시 폴더 삭제 |

## 10. 구현 시 주의사항

### 10.1 기존 코드 재활용

- **찬양의 숲**의 목록 UI, 재생 로직, R2 업로드 Worker를 기반으로 확장
- KV 키 네이밍만 `openroom:` 접두어로 변경하고 구조는 동일하게 유지
- 인증 체계는 기존 로그인 시스템 공유

### 10.2 R2 업로드 처리

- `multipart/form-data`로 수신
- Worker에서 R2에 직접 PUT (presigned URL 방식도 가능하나, 50MB 이하이므로 직접 전송으로 충분)
- 업로드 완료 후 오디오 duration 추출이 필요함 — 클라이언트에서 `Audio` 객체로 duration 측정 후 메타데이터에 포함하여 전송하는 방식 권장

### 10.3 참조 무결성

- 복사/이동/삭제 시 반드시 **폴더 메타데이터**와 **파일 마스터 레코드** 양쪽을 함께 갱신
- KV는 트랜잭션을 지원하지 않으므로, 파일 마스터 레코드를 먼저 갱신한 뒤 폴더 메타데이터를 갱신하는 순서 유지
- 실패 시 고아 참조가 발생할 수 있으므로, Cron Trigger의 고아 파일 정리가 안전망 역할

### 10.4 곡 이름 자동 추출

```javascript
// 파일명에서 확장자 제거
// "주의_사랑이_나를_감싸네.mp3" → "주의 사랑이 나를 감싸네"
function extractDisplayName(filename) {
  return filename
    .replace(/\.[^/.]+$/, '')     // 확장자 제거
    .replace(/[_-]/g, ' ')        // 언더스코어, 하이픈 → 공백
    .trim();
}
```

## 11. 2단계 확장 — 랜덤 라이브 모드 (참고용)

> ⚠️ 1단계 완료 후 구현. 본 문서에서는 데이터 구조 호환성 확보를 위한 참고만 제공.

1단계의 R2 파일 + KV 메타데이터를 그대로 사용하며, 재생 로직만 추가한다.

```
┌──────────────────────────────┐
│  🎵 열린 음악방               │
├──────────────────────────────┤
│  재생 모드                    │
│  [📋 목록]  [📻 라이브]       │  ← 토글 탭
└──────────────────────────────┘
```

- **목록 모드**: 1단계 — 폴더 탐색 + 개인 재생
- **라이브 모드**: 전체 곡 랜덤 셔플, 모든 청취자 동기 재생

라이브 모드 추가 시 필요한 KV 키:
```
openroom:live:now_playing    → 현재곡 r2Key + 시작 timestamp
openroom:live:queue          → 셔플된 대기열
openroom:live:listeners      → 청취자 수 (TTL 기반)
```

1단계에서 이 키들을 사용하지 않도록 네이밍을 분리해 두었으므로 충돌 없이 확장 가능.
